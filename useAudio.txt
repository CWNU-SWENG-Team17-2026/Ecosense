useAudio.ts 동작 요약 (v1.2 — frontend/src/hooks/useAudio.ts 기준, 2026-06)

※ 이 파일은 설계 스니펫이 아니라 현재 구현 요약입니다.
  과거 useAudio.txt의 Uint8Array/getByteTimeDomainData 버전은 폐기됨.

────────────────────────────────────────
핵심 상수
────────────────────────────────────────
- CHART_UPDATE_INTERVAL_MS = 500   … store 차트 갱신 0.5초 간격
- SPIKE_END_QUIET_MS = 1000        … 임계값 이하 1초 연속 시 녹음 종료

────────────────────────────────────────
dB 계산
────────────────────────────────────────
- AnalyserNode.getFloatTimeDomainData (fftSize 8192)
- RMS → 20*log10(rms)
- 마이크: echoCancellation/noiseSuppression/autoGainControl = false

────────────────────────────────────────
스파이크 감지 (sleep 모드만)
────────────────────────────────────────
- backgroundDb: useAudio 로컬 state -65, useNoiseStore 초기값 -60
- 임계값: backgroundDb + 15dB
- 초과 → MediaRecorder(webm/opus) 시작, isSpiking=true
- 미만 → spikeQuietStartRef로 1초 조용하면 stop
- stop 시 duration >= 1초 & chunks 있으면 indexedDB saveSpikeWithBlob + addSpike

────────────────────────────────────────
세션
────────────────────────────────────────
- startAudioMonitoring: sleep 모드면 startNewSession('SLEEP') + saveSession(indexedDB)
- stopAudioMonitoring: saveSession(ended_at) + endCurrentSession()

────────────────────────────────────────
반환 API
────────────────────────────────────────
currentDb, backgroundDb, isMeasuring, isSpiking,
startAudioMonitoring, stopAudioMonitoring,
startAudio / stopAudio (별칭)

────────────────────────────────────────
연동
────────────────────────────────────────
- useNoiseStore (persist: spikes, sessionId, sessionStartTime)
- indexedDB: spikes + sessions
- useSync는 useAudio 내부에서 직접 호출하지 않음 (Settings/SessionHistory 경로)
